0. Beginning

I install the QGIS-LTR 3.40.5-Bratislava

My testing is primarily conducted in QGIS, while all scripts are executed on Ubuntu. In principle, GPU computation is not required.

The road data provided by Kiki, as well as the outputs generated by my code, are imported into QGIS for manual inspection.

The output file from my code can directly import in QGIS with GeoJSON.

1. Overview

This tool takes scanned Melway street map GeoTIFFs as input and automatically detects road changes across different years, producing three categories of output:

  • persisted (paper shows remained) — roads present in both the historical and modern map
  • added — roads that appear in the modern map but not the historical one
  • disappeared — roads present in the historical map that no longer exist today

2. Installation

Requirements

  • Python 3.9+ (3.10 recommended)
  • pip

Steps

# 1. Enter the project root
cd kiki_roadmap_exisit_checker

# 2. Create a virtual environment (recommended)
# or using anaocnda 
python3 -m venv .venv
source .venv/bin/activate      # Linux / macOS
# .venv\Scripts\activate       # Windows

# 3. Install dependencies
pip install -r requirements.txt

If you only need the core pipeline, you can install just the essential packages:

pip install geopandas rasterio shapely opencv-python scikit-image scikit-learn joblib tqdm pandas numpy easyocr

About `mel_roads_m<id>_old_2.geojson`

This file represents the historical road network for a single map sheet, stored as GeoJSON LineStrings. mel_roads_m<id>_old_2.geojson is automatically generated from {MAP}_rgba.tif , where the roads are directly extracted by tracing from the image. As this process is line-based, the resulting geometry has relatively lower precision.

How the “modern roads” source is determined

get_paths() in config.py picks the modern road reference depending on the year:

  • year == "2020" → reads data/melway/2020/mel_roads_m<id>.geojson (the latest baseline)
  • year != "2020" → reads outputs/<next_year>/m<id>/historical_db.geojson (chained from the next year’s output)

When processing historical years, always run newer years first: 2011 → 2006 → 2001.

4. Quick Start (Single Map Sheet)

Run from inside the my-map/ directory:

cd my-map

# Basic run (2020, sheet m003)
python src/main.py --map_id 003 --year 2020

# Skip road extraction if a historical GeoJSON already exists
python src/main.py --map_id 003 --year 2020 --skip_extraction

# Process a historical year (next_year required)
python src/main.py --map_id 003 --year 2011 --next_year 2020

# Custom log directory
python src/main.py --map_id 003 --year 2020 --log_dir ./logs

8. Module Reference (src/)

This section describes what each Python file in my-map/src/ does and which parts a developer is most likely to need to modify.

main.py — Pipeline entry point

Parses CLI arguments (--map_id, --year, --next_year, --skip_extraction), then calls each stage in order. This is the only file that knows the full execution sequence.

config.py — Central configuration

Defines every tunable parameter in one place and the get_paths() function that resolves all file paths for a given (map_id, year, next_year) combination.

tif_converter.py — Palette TIF → RGBA TIF (Step 1)

Reads a single-band indexed-colour GeoTIFF and expands the palette into a 4-band RGBA GeoTIFF. This is always the first step because all later image-processing works on RGB pixel values.

Key function: palette_to_rgba(src_path, dst_path)

When to modify: if the source TIFs use a different colour encoding (e.g. already RGB, or 16-bit).

road_extractor.py — Colour masking + skeletonisation (Step 2)

The most complex image-processing stage:

  1. Reads the RGBA TIF and converts it to BGR.
  1. For each road class (residential, trunk, primary, tertiary, motorway, unclassified), applies a colour-tolerance mask and morphological cleanup.
  1. Skeletonises each mask with skimage.morphology.skeletonize.
  1. Traces the skeleton pixels into polyline segments, splitting at junctions.
  1. Filters segments against a modern-road spatial prior (proximity threshold MODERN_BUFFER_DIST).
  1. Smooths and exports as mel_roads_m<id>_old_2.geojson.

Key function: run(paths, cfg) — called by main.py.

feature_extractor.py — HIRONEX feature extraction + RF prediction (Step 3)

For each modern road segment:

  1. Samples RGB pixel values along the road centreline and on cross-sections perpendicular to it.
  1. Computes three HIRONEX visual features: Var_line (along-line pixel variation), Edge_contrast (road-vs-background colour contrast), and ROI (cross-section gradient integral).
  1. Loads the pre-trained Random Forest model and predicts Pred (1 = road visually present in the TIF, 0 = absent).

Returns a GeoDataFrame df_feat used by all later comparison stages.

Key function: extract_features(src, modern_path, geo_crs, metric_crs)

loop_detector.py — Circular loop detection (Step 4a)

Identifies road segments that form closed circular loops (e.g. roundabouts, cul-de-sac turning circles). If any segment in a loop group is predicted as Pred=0, the whole group is forced to Pred=1, because a partially-missing ring is geometrically implausible.

Key functions:
- detect_circular_loops(df_feat) — finds loop groups.
- apply_loop_label_propagation(df_feat, loop_groups) — flips Pred for affected segments.

connectivity_fixer.py — Connectivity-based label promotion (Step 4b)

Examines the road network topology. A contiguous group of Pred=0 segments that bridges two distinct Pred=1 endpoints (one at each end) is promoted to Pred=1 — because a gap in the middle of a real road is implausible.

Key function: fix_connectivity(df_feat)

road_comparator.py — Historical vs modern comparison (Step 5)

  1. Loads the historical GeoJSON and fixes any invalid CRS.
  1. Buffers each historical segment by BUFFER_DIST metres and checks how much of it overlaps with Pred=1 modern roads.
  1. Assigns status = "both" if overlap ≥ MIN_OVERLAP_RATIO, else "disappeared".
  1. Enriches "both" rows with attributes from df_feat (Step 5.5).
  1. Classifies modern roads into persisted / added.
  1. Exports persisted.geojson, added.geojson, disappeared.geojson, and historical_all.geojson.

Key functions: compare_historical_modern(), enrich_historical_with_features(), classify_changes(), export_results(), build_and_export_historical_db()

ocr_filter.py — Text artefact removal (Step 6)

Runs EasyOCR on the RGBA TIF to detect text regions (road labels, suburb names, etc.). Any disappeared road whose geometry overlaps a text polygon by more than OCR_RATIO_TH is removed — these are skeleton artefacts caused by dark map text being misdetected as roads.

If easyocr is not installed the filter is silently skipped.

Key function: remove_text_artefacts(historical, rgba_tif_path, ...)

post_processor.py — Final cleanup (Step 8)

Two independent cleanup passes on historical_db before final export:

A. Parallel + shorter filter — if a disappeared segment runs parallel (within PARALLEL_ANGLE_TH_DEG) to a non-disappeared neighbour and is shorter, it is dropped as a likely duplicate extraction artefact.

B. Residential-adjacent filter — disappeared roads of certain classes (trunk, primary, tertiary, residential) that are adjacent to a residential road are removed, as these typically represent mis-classified service ramps or alley artefacts.

Key function: clean_historical_db(historical_db)

logger.py — Logging setup

Sets up a module-level logger that writes to both stdout (INFO level) and a timestamped file in logs/ (DEBUG level). Each pipeline invocation creates a fresh log file.

Key function: get_logger(name) — import and call this at the top of any new module.

utils.py — Shared helpers

Utility functions used across multiple modules:

9. Troubleshooting

RGBA TIF not found

FileNotFoundError: m007_rgba.tif not found

tif_converter.palette_to_rgba() generates this file automatically on the first run. If the original palette TIF data/melway/<year>/m<id>.tif is missing, place it in the correct path and re-run.

Historical road output is empty (0 features)

Possible causes:

  1. Colour mismatch → increase COLOR_TOLERANCE (e.g. 15–20)
  1. Paths too short → decrease MIN_PATH_LEN_PIX (e.g. 5)
  1. All segments filtered by the modern-road proximity check → increase MODERN_BUFFER_DIST

CRS warning on historical GeoJSON

⚠️ Fixing: historical CRS is invalid (LOCAL_CS), forcing EPSG:7844

This is expected behaviour. When mel_roads_m<id>_old_2.geojson has incomplete CRS metadata, the pipeline automatically falls back to EPSG:7844. It does not affect the results.

OCR is removing too many roads

Adjust OCR_RATIO_TH (default 0.4):
- Raise it (e.g. 0.6) → only remove roads with more than 60% overlap with text regions
- Raise OCR_CONF_TH (e.g. 0.6) → only trust high-confidence OCR detections

Missing modern road reference for historical years

FileNotFoundError: outputs/2011/m007/historical_db.geojson

Processing 2006 requires the 2011 outputs to already exist. Run years in order: 2011 first, then 2006, then 2001.

Wrong model path

Confirm that models/rf_model_20251110.joblib exists. If your model file has a different name, update config.py:

MODEL_FILENAME = "rf_model_20251110.joblib"  # ← set to your actual filename

Web Demo

https://github.com/anonymous-road-atlas/melway/

An interactive web map that visualises how Melbourne’s road network changed across four decades, using Melway historical map data compared against a modern road network.

Quick Start

Because the page fetches local GeoJSON files via fetch(), you must serve it through an HTTP server — browsers block direct file:// access.

Option 1 — Python (no install required)

cd my-map
python3 -m http.server 8080

Open your browser at http://localhost:8080

Option 2 — Node.js

npx serve my-map

Option 3 — VS Code Live Server

Right-click index.html → Open with Live Server.

Interface Overview

Controls

Year Buttons

Switches the active dataset to a different historical snapshot.

ButtonData year
2001Earliest snapshot
2006—
2011—
2020Default (loaded on startup)

Clicking a year button:
1. Removes all currently rendered road layers from the map.
2. Reads melway_outputs/<year>/blocks.json to discover which map blocks exist.
3. Loads historical_db_final.geojson (grey base layer) for every block — always visible.
4. Loads added, persisted, and disappeared GeoJSON for every block — shown only if the corresponding toggle is active.

The map viewport is not reset when switching years, so you can compare the same area across time.

Layer Toggle Buttons

These three buttons independently show or hide a road change category.

ButtonColourMeaning
AddedBlue #007bffRoad appears in the selected year but not in the previous snapshot
RemainedGreen #40be7fRoad exists in both the previous and selected year
DisappearedRed #ff0000Road existed historically but is absent in the selected year

The grey base layer (Historical DB) is always visible and cannot be toggled off.

Clicking a toggle button switches it on or off without moving the map or reloading data.

Focus Area Buttons

Instantly fly the camera to a pre-defined region of Melbourne.

NorthNorth Melbourne (around m003)

These use Leaflet’s flyToBounds() for a smooth animated pan/zoom.

Data Structure

melway_outputs/
└── <year>/                        # 2001 | 2006 | 2011 | 2020
    ├── blocks.json                # ["m003", "m007", "m043", ...]
    └── <block>/                   # one directory per map sheet
        ├── historical_db_final.geojson   # base road network (always shown)
        ├── added.geojson
        ├── persisted.geojson
        └── disappeared.geojson

blocks.json controls which blocks are loaded for each year — missing blocks or missing files are silently skipped, so partial datasets work fine.

How the Loading Works (Technical)

  1. loadYear(year) is called on startup with "2020".
  1. It fetches melway_outputs/2020/blocks.json → gets an array of block IDs.
  1. For each block, six fetch() calls fire in parallel:
    • historical_db_final.geojson → rendered immediately, added to the map.
    • added, persisted, disappeared → stored in memory, added to the map only if the toggle is active.
  1. After 300 ms, the map auto-fits its viewport to the loaded base layers (only on first load).
  1. Subsequent loadYear() calls clear the previous layers and repeat steps 2–3. The viewport is not changed.

Adding a New Year

  1. Create melway_outputs/<new_year>/.
  1. Add blocks.json:
    ["m003", "m007"]
  1. Add the four GeoJSON files for each block.
  1. Register the year in index.html (line ~196):
    const years = ["2001", "2006", "2011", "2020", "<new_year>"];
